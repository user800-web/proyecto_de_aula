I. Introducción a las Aplicaciones Web

"Inicialmente la web era simplemente una colección de páginas estáticas, documentos, etc., que podían consultarse o descargarse. El siguiente paso en su evolución fue la inclusión de un método para confeccionar páginas dinámicas que permitiesen que lo mostrado fuese dinámico (generado o calculado a partir de los datos de la petición). Dicho método fue conocido como CGI (common Gateway interface) y definía un mecanismo mediante el cual podíamos pasar información entre el servidor HTTP y programas externos. Los CGI siguen siendo muy utilizados, puesto que la mayoría de los servidores web los soportan debido a su sencillez. Además, nos proporcionan total libertad a la hora de escoger el lenguaje de programación para desarrollarlos. El esquema de funcionamiento de los CGI tenía un punto débil: cada vez que recibíamos una petición, el servidor web lanzaba un proceso que ejecutaba el programa CGI. Como, por otro lado, la mayoría de CGI estaban escritos en algún lenguaje interpretado (Perl, Python, etc.) o en algún lenguaje que requería run-time environment (VisualBasic, Java, etc.), esto implicaba una gran carga para la máquina del servidor. Además, si la web tenía muchos accesos al CGI, esto suponía problemas graves.

II. Evolución de la WEB

Web 1.0

Apareció en 1960, se caracterizaba por ser estática, secuencial, centralizada, de solo lectura y no interactiva; solo se podía consumir contenido de la web sin poder interactuar con ella como si fuera un libro. En la época de la Web 1.0 el navegador más usado era ELISA. No fue hasta que llegó HTML que experimentó una mejora en la experiencia con una mejor estructura y más atractivo para leer. Sin embargo, aún con la llegada de HTML las páginas web no dejaron de ser estáticas hasta que llegó la Web 2.0

Web 1.5

La web 1.5 es una web que aparece entre la Web 1.0 y la 2.0. Esta web aparece entre los años 1997 y 2003. Desde esta web ya es posible hacer páginas “dinámicas” ya que permite al webmaster y al cliente modificar la información de la página, modificando la base de datos.

Web 2.0

La Web 2.0 o también conocida como la Web Social empieza a ser planificada en 2001 y no es hasta 2004 que surge “oficialmente”. La Web 2.0 se caracteriza por ser la que dio vida a las páginas dinámicas. De ésta nacieron las wikis, foros, blogs, redes sociales, etc. El objetivo de la Web 2.0 fue simple: ser una web que permita a las personas colaborar e interactuar entre sí para mejorar el sitio web. Es por esto que es llamada la Web Social

Web 3.0

La Web 3.0 también conocida como Web Semántica, surge en 2006 y se vuelve operativa en 2010. Si la Web 2.0 estaba enfocado en lo social, entonces la Web 3.0 está enfocado en la personalización. ¿A qué se refiere esto? Bien, la Web 3.0 tiene la particularidad de qué ahora las páginas webs pasan a ser semánticas, lo que significa que ahora pueden interpretar el lenguaje natural. Este nuevo método de interpretación nos da la ventaja de que ahora las aplicaciones o páginas webs nos recomienden cosas que pueden ser de nuestro agrado. ¿Cómo lo hace? Bien, esto es debido a que desde la Web 3.0 las páginas webs empiezan a funcionar a través de servidores en la nube. En estos servidores quedan registrados las acciones de los usuarios de las páginas webs. También es gracias a estos servidores que ahora se puede acceder a las páginas webs desde cualquier dispositivo inteligente. Una página web usa los datos de tus opiniones, comentarios y búsquedas más populares para recomendarte cosas relacionadas, toda esta información es sacada de los servidores en la nube

Web 4.0

La Web 4.0 o mejor conocida como Web Predictiva surge en 2016. La Web 3.0 fue la punta del iceberg, la Web 4.0 ya no solo busca recomendar, ahora busca responder con soluciones ¿Cómo lo hará? La Web 4.0 se ve caracterizada por el uso de inteligencia artificial y bots. Estos usarán toda la información del entorno y la web para encontrar la mejor solución a las peticiones del usuario. La Web 4.0 también tiene otra particularidad, ahora es posible usar comandos de voz para hacer que los bots realicen acciones. Por ejemplo: para pedir un taxi podríamos decir “Siri, pide un taxi para esta dirección”; lo que el bot hará es interpretar la orden, sabrá que hay que pedir un taxi, así que buscará como hacerlo en la web y con eso cumplirá la orden. Cabe destacar que en este ejemplo el bot sacará la dirección del gps del dispositivo del usuario. También esta el tema de que debido a la inteligencia artificial ahora las páginas webs ya no solo recomendarán cosas parecidas a lo que has buscado, sino que también empezarán a recomendarte cosas que podrían gustarte. Haciendo alarde de su capacidad predictiva
