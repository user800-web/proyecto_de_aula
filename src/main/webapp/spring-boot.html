Spring Framework

Una pequeña historia

Spring Framework fue creado en 2003 por Rod Johnson, autor de J2EE Development sin EJB (Wrox Publishing, 2004). Spring Framework fue la respuesta a todas las complejidad que tenían las especificaciones J2EE en ese momento. Hoy ha mejorado, pero usted necesita tener una infraestructura completa para ejecutar ciertos aspectos del ecosistema J2EE. Podemos decir que Spring es una tecnología complementaria a Java EE. El marco de primavera integra varias tecnologías, como Servlet API, WebSocket API, concurrencia utilidades, API de enlace JSON, validación de beans, JPA, JMS y JTA/JCA. Spring Framework admite la inyección de dependencia y la anotación común especificaciones que facilitan el desarrollo.

Spring Boot

Spring Boot simplifica la forma en que desarrollamos porque facilita la creación Aplicaciones basadas en Spring listas para producción que puede "simplemente ejecutar". Tu lo descubrirás que con Spring Boot, puede crear aplicaciones independientes con un servidor integrado (Tomcat por defecto, o Netty si está utilizando los nuevos módulos web reactivos), haciendo ellos 100% ejecutables y desplegables.


Puede encontrar el proyecto Spring Boot en https://projects.spring.io/spring-boot/. Hay documentación muy extensa en https://docs.spring.io/spring-boot/docs/ actual/referencia/htmlsingle/.

Spring Boot al rescate

Para crear aplicaciones Spring es necesario conocer todas las técnicas de configuración y/o requisitos para la tecnología. Se requieren muchos pasos para ejecutar incluso el aplicación de primavera más simple. Hace cuatro años, el equipo de Spring lanzó la primera versión beta, que tuve la suerte de probar.


Los resultados fueron asombrosos. Ahora, con más características añadidas a la tecnología, realmente se ha convertido en la forma “de facto” de crear aplicaciones Spring. Spring Boot facilita la creación de aplicaciones listas para la empresa. Si echa un vistazo a la página web del proyecto Spring Boot, una declaración que dice: Absolutamente sin generación de código y sin requisitos para la configuración de XML.


Quizás se pregunta cómo puede crear aplicaciones Spring y ejecutarlas sin ningún tipo de configuración. El contenedor Spring al menos necesita saber cómo conectar sus clases, ¿Correcto? O el contenedor Spring necesita saber cómo usar la tecnología que agregó a tu aplicación Bueno, no te preocupes. Te contaré todos los secretos detrás de esta increíble tecnología.
