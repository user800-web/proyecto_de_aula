JSP

JavaServer Pages (JSP) es otra tecnología Java para desarrollar aplicaciones web. JSP fue lanzado durante el tiempo en que la tecnología de servlet había ganado popularidad como uno de los mejores tecnologías disponibles. Sin embargo, JSP no pretende reemplazar los servlets. De hecho, JSP es una extensión de la tecnología de servlet, y es una práctica común usar servlets y páginas JSP en el mismo aplicaciones web.

Crear páginas JSP es tan fácil que puede escribir aplicaciones JSP sin mucho conocimiento de la API subyacente. Sin embargo, si desea ser un buen programador web de Java, debe conocer JSP y servlets. Incluso si usa solo páginas JSP en sus aplicaciones web Java, comprender los servlets sigue siendo muy importante, [1].

El código fuente de una página JSP puede contener:

  1. Directivas: Indican información general de la página, como puede ser importación de clases, página a invocar ante errores, si la página forma parte de una sesión, etc.
  2. Declaraciones: Sirven para declarar métodos o variables.
  3. Scriptlets: Código Java embebido.
  4. Expresiones: Expresiones Java que se evalúan y se envían a la salida.
  5. Tags JSP: Etiquetas especiales que interpreta el servidor.

Elementos de JSP

Código Java:Podemos insertar código Java dentro de JSP de tres formas: Expresiones, scriptlets y declaraciones.

Directivas:Las directivas son elementos que proporcionan información al motor JSP, e influirán en la estructura del servlet generado. Hay tres tipos de directivas: page, taglib e include.

Acciones:Las acciones tienen la forma <jsp:accion [parámetros]/>, y tienen diversos usos, entre los que destacan la inclusión de páginas y transferencia de control, [2].
